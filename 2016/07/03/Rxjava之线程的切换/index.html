Rxjava之线程的切换

发表于 2016-07-03   |  


文档解释

  • subscribeOn : 指示Observable在一个特定的调度器上运转.它同样会在那个调度器上给观察者发通知。
  • ObserveOn : 指示Observable在一个指定的调度器上给观察者发通知.

不是很明白?那就看例子,一个个写出来

还是要从使用看起

最简单的一个小例子

E/–>>: create – main
E/–>>: onNext – main

在一个线程里面的话

E/–>>: create – Thread-5297
E/–>>: onNext – Thread-5297

可以看到 如果没有任何操作符处理的话,默认是执行在当前线程的

subscribeOn

在主线程里指定在新线程里执行

E/–>>: create – RxNewThreadScheduler-1
E/–>>: onNext – RxNewThreadScheduler-1

在线程里面用subscribeOn指定主线程

E/–>>: create – main
E/–>>: onNext – main

只有subscribeOn操作符的情况下.发射数据与接收数据都是在subscribeOn指定的线程里.与文档介绍的一致

ObserveOn

E/–>>: create – main
E/–>>: map no change – main
E/–>>: map no change – RxNewThreadScheduler-1
E/–>>: onNext RxNewThreadScheduler-1

E/–>>: create – Thread-5576
E/–>>: map no change – Thread-5576
E/–>>: map no change – main
E/–>>: onNext main

可以看到ObserveOn影响的是下面的操作

SubscribeOn 与 OnserveOn

create – RxNewThreadScheduler-1
E/–>>: map no change – RxNewThreadScheduler-1
E/–>>: map no change – RxNewThreadScheduler-1
E/–>>: map no change – main
E/–>>: onNext main

create – RxNewThreadScheduler-1
E/–>>: map no change – RxNewThreadScheduler-1
E/–>>: map no change – main
E/–>>: map no change – main
E/–>>: onNext main

当SubscribeOn操作符在ObserveOn的上面.ObserveOn操作符下面的操作所在的线程由ObserveOn处理,当SubscribeOn操作符在ObserveOn的下面.ObserveOn操作符下面的操作所在的线程由ObserveOn处理.

结果

上面两种写法的结果是一样的
